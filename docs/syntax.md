# Fragmos Block Language — Спецификация синтаксиса `.frg`

## Структура файла

Файл `.frg` состоит из двух частей:

```
CONFIG:
    ключ = значение
    ...
END

START "Начало"
...
STOP "Конец"
```

Секция `CONFIG` — опциональна. Если её нет, используются значения по умолчанию.

---

## Секция CONFIG

```
CONFIG:
    gap_y              = 40     # вертикальный зазор между элементами
    if_branch_vgap     = 15     # зазор: низ ромба → верх первого блока ветки
    if_branch_min_gap  = 40     # минимальный зазор между bbox-ами веток IF
    while_corridor_base = 80    # ширина коридора возвратной стрелки (глубина 0)
    while_corridor_step = 20    # уменьшение коридора на каждый уровень вложенности
    while_corridor_min  = 30    # минимальная ширина коридора
    while_back_turn_gap = 20    # зазор: низ последнего блока → перемычка возврата
    while_back_top_gap  = 15    # зазор: линия коридора → верх ромба
    show_bbox          = true   # рисовать цветные bounding box (true / false)
END
```

Все параметры опциональны — не указанные берутся из DEFAULT_CFG.

## Блоки

### Простые блоки

| Ключевое слово | Тип (`type`) | Фигура |
|---|---|---|
| `START "текст"` | `start` | Скруглённый прямоугольник |
| `STOP "текст"` | `stop` | Скруглённый прямоугольник |
| `EXEC "текст"` | `execute` | Прямоугольник |
| `PROCESS "текст"` | `process` | Прямоугольник с двойными краями |
| `IO "текст"` | `io` | Параллелограмм |
| `LOOP_START "текст"` | `loop_limit_start` | Трапеция (шапка цикла ГОСТ) |
| `LOOP_END "текст"` | `loop_limit_end` | Трапеция перевёрнутая (подвал цикла ГОСТ) |

### IF — ветвление

```
IF "условие"
  YES:
    # блоки ветки "Да"
  NO:
    # блоки ветки "Нет"
END
```

- Метки всегда фиксированы: **Да** (правая ветка) / **Нет** (левая ветка).
- Допустимо указать только `YES:` или только `NO:` — вторая ветка будет прямой линией.
- Тело каждой ветки может содержать любые блоки, включая вложенные `IF`, `WHILE`, `FOR`.

### WHILE — цикл с предусловием

```
WHILE "условие"
  # блоки тела цикла
END
```

- Ромб условия сверху.
- Возвратная стрелка идёт по левому коридору.
- Выход из цикла (ветка "Нет") уходит вправо.

### FOR — цикл (шестиугольник)

```
FOR "параметры цикла"
  # блоки тела цикла
END
```

- Шестиугольник вместо ромба.
- Логика коридоров и возврата — аналогична `WHILE`.

## Правила синтаксиса

```
1. Текст блока — всегда в двойных кавычках "...".
   Кавычки внутри текста экранируются: \"

2. Комментарии — строки начинающиеся с #. Игнорируются парсером.

3. Отступы — 2 или 4 пробела. Только для читаемости,
   структуру задают ключевые слова, а не отступы.

4. Ключевые слова — регистронезависимы (start = START = Start).

5. Каждый файл должен начинаться с START и заканчиваться STOP.

6. Пустых END-блоков быть не должно: тело WHILE/FOR — минимум 1 блок.
```

## Пример полного `.frg` файла

```
CONFIG:
    gap_y     = 50
    show_bbox = false
END

START "Начало"

IO "Ввод: массив a[], n"

EXEC "i = 0, sum = 0"

WHILE "i < n"
  IF "a[i] > 0"
    YES:
      EXEC "sum = sum + a[i]"
    NO:
      EXEC "neg_count = neg_count + 1"
  END
  EXEC "i = i + 1"
END

IO "Вывод: sum"

STOP "Конец"
```

## Соответствие → структура `nodes`

Парсер должен преобразовать `.frg` в список словарей:

```python
# START "Начало"
{'type': 'start', 'value': 'Начало'}

# STOP "Конец"
{'type': 'stop', 'value': 'Конец'}

# EXEC "x = 5"
{'type': 'execute', 'value': 'x = 5'}

# PROCESS "Обработка"
{'type': 'process', 'value': 'Обработка'}

# IO "Ввод x"
{'type': 'io', 'value': 'Ввод x'}

# LOOP_START "i = 1 до 10"
{'type': 'loop_limit_start', 'value': 'i = 1 до 10'}

# LOOP_END "i = 1 до 10"
{'type': 'loop_limit_end', 'value': 'i = 1 до 10'}

# IF блок:
{
    'type': 'if',
    'value': 'x > 0',
    'children':      [...],   # ветка YES
    'else_children': [...],   # ветка NO
}

# WHILE блок:
{
    'type': 'while',
    'value': 'i < n',
    'children': [...],
}

# FOR блок:
{
    'type': 'for_default',
    'value': 'i in range(n)',
    'children': [...],
}
```

## Пример: файл выше → nodes

```python
cfg = {
    'gap_y': 50,
    'show_bbox': False,
    # остальные параметры из DEFAULT_CFG
}

nodes = [
    {'type': 'start', 'value': 'Начало'},
    {'type': 'io', 'value': 'Ввод: массив a[], n'},
    {'type': 'execute', 'value': 'i = 0, sum = 0'},
    {
        'type': 'while',
        'value': 'i < n',
        'children': [
            {
                'type': 'if',
                'value': 'a[i] > 0',
                'children': [
                    {'type': 'execute', 'value': 'sum = sum + a[i]'},
                ],
                'else_children': [
                    {'type': 'execute', 'value': 'neg_count = neg_count + 1'},
                ],
            },
            {'type': 'execute', 'value': 'i = i + 1'},
        ],
    },
    {'type': 'io', 'value': 'Вывод: sum'},
    {'type': 'stop', 'value': 'Конец'},
]
```
