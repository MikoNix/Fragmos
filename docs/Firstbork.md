# Введение в Bock

Bock — псевдоязык для описания блок-схем. Нейросеть переводит любой исходный код в Bock, а интерпретатор Fragmos превращает его в визуальную схему.

## Структура программы

Каждая блок-схема начинается с объявления функции:

```bock
function [NAME] {
    // команды
}
```

Где `NAME` — название функции из исходника. Если в исходнике нет имени функции — используем `main`.

Каждая функция строится в отдельном `block.json`. Первым блоком всегда является блок **Начало**.

---

## Переменные

Переменные не требуют объявления типа. Любое выражение с `=` превращается в блок действия, в текст которого записывается полное выражение.

```bock
function example {
    i = 10 + 456; // блок будет содержать: i = 10 + 456
    b = i - 10;   // блок будет содержать: b = i - 10
}
```

---

## Ввод и вывод

Для ввода от пользователя: `input >> ""`.  
Для вывода: `output >> ""`.  
Всё, что указано внутри кавычек, записывается в соответствующий блок.

```bock
function input_and_output {
    input >> "i";       // одиночный ввод
    input >> "a,b,c";   // множественный ввод — перечисляем через ,
    output >> "i";      // одиночный вывод
    output >> "a,b,c";  // множественный вывод — перечисляем через ,
}
```

Как выглядят эти блоки визуально — смотри [[Блоки и их отрендеренные версии]].

---

## Комментарии

Комментарии пишутся после `//` и не попадают в блок-схему.

```bock
function nothing_to_see_here {
    // это комментарий, он не отобразится в схеме
}
```

---

## Пример: Hello World

```bock
function Hewoo_world {
    output >> "Hewoo world";
}
```

---

## Пример: два слагаемых

```bock
function two_plus_two {
    i = 10;
    b = 10;
    output >> i + b;
}
```
